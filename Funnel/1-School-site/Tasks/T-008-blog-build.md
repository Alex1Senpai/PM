# T-008 — Построить блог: SEO-восстановление и воронка прогрева

## Контекст

Старый сайт Oxbridge содержал блог с образовательными статьями, которые ранжировались в Google. После редизайна эти страницы исчезли, но Google продолжает отправлять на них трафик — пользователи попадают на мёртвые страницы (100% exit rate, <8 сек сессии). Подтверждено через Clarity.

Блог выполняет две функции: восстановление потерянного SEO-трафика + станция прогрева лидов. Родители, которые ищут образовательный контент (интервальное повторение, мнемоника, педагогические методы), находят Oxbridge в процессе исследования — до того, как начнут искать школу. Это верхняя часть воронки.

Контент пишет Claude, утверждает Bekhruz. Расписание публикаций: 15-е и 30-е каждого месяца. Все одобренные посты публикуются на сайт. В Telegram уходят 5 постов за раз (остальные хранятся в банке).

**Стадия воронки:** 1-School-site
**Архив старого сайта:** доступен — использовать как источник для первых постов.

---

## MVP

Минимальная версия, которую можно задеплоить и показать Bekhruz:

- Роут `/blog/[slug]` работает для русских постов
- 5 постов из архива опубликованы и доступны (русский язык)
- Каждый пост содержит: заголовок, тело, дата публикации, CTA-блок внизу
- CTA-блок внизу: кнопка «Записаться на экскурсию» с якорем на форму
- Страница `/blog` (индекс) — список постов с заголовком и датой, без фильтров
- HTML-структура каждого поста совместима с Telegram InstantView (стандартная семантическая разметка: `<article>`, `<h1>`, `<time>`, `<p>` — без кастомных JS-зависимых компонентов в теле статьи)

## Finished State

Полная версия, готовая к production:

**Контент и структура**
- Поддержка всех языков: `/blog/[slug]` (RU), `/uz/blog/[slug]`, `/en/blog/[slug]`, `/tr/blog/[slug]`
- Индекс `/blog` с фильтрацией по категориям (тег/тема): минимум 3 категории на старте (Методика обучения, Поступление, Жизнь в школе)
- Каждый пост: заголовок, тело, дата, автор (по умолчанию «Команда Oxbridge»), обложка (если есть), теги

**CTA-система внутри постов**
- Нижний CTA-блок (обязателен на каждом посте): «Записаться на экскурсию» → `https://ig.me/m/oisuzbekistan?text=САЙТ`
- Средний CTA-блок (1–2 раза в длинном посте): внутренняя ссылка на релевантную страницу программы или кампуса. Позиция определяется в тексте поста — Claude проставляет метку `[MID-CTA: /futurum]`, Alex рендерит как компонент.

**Telegram InstantView**
- HTML-структура поста должна корректно парситься IV-шаблоном Timur'a (T-007)
- Тело поста — чистый HTML без клиентского рендеринга

**Контентный workflow**
- Черновики хранятся в `/blog/drafts/` (или аналог в CMS) со статусом: `draft` / `approved` / `published` / `scheduled`
- Опубликованные посты видны на сайте сразу после смены статуса на `published`
- Telegram-публикация — отдельный флаг `telegram: true/false`. Не все `published` посты уходят в Telegram автоматически.

**Технические требования**
- Открытые Graph теги на каждом посте (og:title, og:description, og:image)
- Canonical URL проставлен корректно для каждой языковой версии
- sitemap.xml обновляется автоматически при публикации нового поста
