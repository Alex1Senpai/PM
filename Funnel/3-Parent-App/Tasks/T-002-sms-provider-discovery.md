# T-002 — Discovery: выбрать SMS-провайдера для OTP в Узбекистане

## Контекст
Parent App использует Phone OTP через SMS как основной метод авторизации (D-004). Для этого нам нужен надёжный SMS-провайдер в Узбекистане. Без выбранного провайдера Алекс не может перевести AuthService с мокового на реальный.

Это **discovery-задача**. Ты не выбираешь сам — ты собираешь факты и приносишь Bekhruz рекомендацию. Решение принимает он.

**Спецификация:** [[../Specs/payments]] (косвенно — auth для платежей)
**Решение, к которому относится:** [[../Decisions/D-004-auth-model]]

**Стадия воронки:** 3-Parent-App

### Итоговое сравнение для OTP

**Eskiz** - самая дешевая (95 сум), хорошая документация, но требует согласования отправителя.
**PlayMobile** - 10 лет опыта, используют 2500+ кампаний, хорошая надежность.
**SMS.uz** - бесплатные буквенные отправители, оплата только за доставленные, удобный API, 500 сум за смс. Нужно договориться о встрече, чтобы узнать больше информации.
GetSMS - дешёвый но нестабильный вариант. Публичных отзывов нет, плохая документация.

### Рекомендация

Рассматривались: Eskiz, PlayMobile, SMSUZ и GetSMS.

По итогам сравнения наиболее подходящим вариантом для сервисных SMS является Eskiz.

Причины:

- самая низкая стоимость сервисных SMS — 95 сум;
- удобная и прозрачная тарификация;
- хороший API для интеграции;
- оптимален для OTP, кодов подтверждения и уведомлений;
- выгоден при больших объёмах отправки SMS.

Альтернативы:

- PlayMobile — больше подходит для рекламных рассылок;
- GetSMS — подходит для сложных интеграций и гибкой настройки;
- SMSUZ — не рекомендуется из-за высокой стоимости SMS.

Итог:  
Для сервисных SMS рекомендуется использовать Eskiz как основной SMS-провайдер.

## Завершение
> Собрал информацию по основным SMS-провайдерам в Узбекистане и сравнил их по цене, API, удобству интеграции и надёжности.  Для OTP и сервисных SMS наиболее подходящим вариантом выглядит Eskiz. Остальные провайдеры можно рассматривать как запасные варианты.  Некоторые детали (тестовые доступы и дополнительные условия) ещё нужно уточнять напрямую у провайдеров во время интеграции.
