# T-002 — Discovery: выбрать SMS-провайдера для OTP в Узбекистане

## Контекст
Parent App использует Phone OTP через SMS как основной метод авторизации (D-004). Для этого нам нужен надёжный SMS-провайдер в Узбекистане. Без выбранного провайдера Алекс не может перевести AuthService с мокового на реальный.

Это **discovery-задача**. Ты не выбираешь сам — ты собираешь факты и приносишь Bekhruz рекомендацию. Решение принимает он.

**Спецификация:** [[../Specs/payments]] (косвенно — auth для платежей)
**Решение, к которому относится:** [[../Decisions/D-004-auth-model]]

**Стадия воронки:** 3-Parent-App

## Что нужно сделать

1. **Найти все актуальные SMS-провайдеры в Узбекистане.** Минимум, который должен быть рассмотрен:
   - **Eskiz** (eskiz.uz)
   - **PlayMobile** (playmobile.uz)
   - **SMS.uz** или аналоги
   - Любые другие, которые ты найдёшь
   - Возможно — местные банковские шлюзы, если они предоставляют OTP-сервис

2. **По каждому провайдеру собрать данные** в одну таблицу (внутри этого файла, в разделе «Результат»). Колонки:
   - Название
   - Веб-сайт и контакты
   - Тип API (REST? SOAP? Webhook?)
   - Документация — есть/нет, на каком языке, насколько полная
   - Стоимость одного SMS (в UZS или USD)
   - Минимальная предоплата / абонентская плата
   - Сертификации (если есть — например, для финансовых SMS)
   - Поддержка отправки кириллических символов
   - Скорость доставки (если в документации указано)
   - Reliability — есть ли отзывы, статистика доставляемости
   - Способ оплаты для нас (юрлица в Узбекистане)

3. **Запросить тестовые доступы** у 2–3 наиболее подходящих провайдеров. Если можно зарегистрироваться и получить тестовый ключ — сделай это.

4. **Тестово отправить 5–10 SMS** через тестовые ключи на свой номер. Засеки время доставки. Запиши результаты.

5. **Составить рекомендацию.** Один параграф: «Я рекомендую X, потому что Y. Альтернатива — Z в случае отказа X.» Bekhruz прочитает и решит.

## Что уже есть / на что можно опираться
- D-004 описывает требования к авторизации
- Бюджет: до $60/месяц на SMS при ~1500 родителях × 2 авторизации/месяц. Этим можно прикинуть, что нам нужен провайдер с ценой около $0.02/SMS или ниже

## План тестирования
- **Ручные проверки:**
  - Отправить SMS на свой номер с тестового ключа каждого провайдера
  - Замерить время от вызова API до получения SMS на устройстве
  - Проверить, что текст SMS пришёл корректно (кириллица, цифры)
- **Граничные случаи:**
  - Что происходит, если номер невалиден? Провайдер возвращает ошибку или просто молчит?
  - Что происходит при rate-limit? Есть ли защита?
  - Что происходит, если SIM-карта оператора X — все ли провайдеры доставляют до всех операторов?
