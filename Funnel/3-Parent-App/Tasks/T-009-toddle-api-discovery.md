# T-009 — Discovery: что Toddle отдаёт через API для посещаемости и student-данных

## Контекст
По _Hub.md, Toddle сегодня — источник истины для учеников, классов, посещаемости, оценок и расписаний. Parent App в v1 читает посещаемость из Toddle (через наш backend), а в долгосроке — student-данные (привязка ребёнка к семье).

К 2027–28 Toddle планируется заменить нашей собственной LMS. Поэтому интеграция должна быть **изолирована** — мы пишем адаптер, который потом подменяется. Discovery нужно сделать сейчас, чтобы понимать **границы возможного** и не строить scope, который Toddle не поддерживает.

Это discovery-задача. Тимур приносит факты. Спецификация `Specs/attendance.md` пишется Bekhruz на основании этих фактов.

**Спецификация:** [[../Specs/attendance]] — *появится после T-009*
**Связанные решения:**
- [[../Decisions/D-007-1c-sync-model]] — описывает тот же шаблон pull для cold-данных
**Стадия воронки:** 3-Parent-App

## Что нужно сделать

1. **Получить доступ к Toddle от Bekhruz** — admin/integration account, чтобы видеть API doc и пробовать вызовы.

2. **Найти и прочитать документацию Toddle API.** Если она не публичная — запросить у Toddle support. Скорее всего, у них есть partner/integration plan.

3. **Ответить на 7 вопросов:**
   1. Есть ли вообще REST/GraphQL API? Или только webhooks? Или только UI-экспорт?
   2. Какая авторизация? (API key? OAuth? Per-school token?)
   3. Какие endpoints отдают посещаемость? Какой формат (per student per day? per class per day?)
   4. Какова задержка между «учитель пометил посещаемость» и «это доступно через API»?
   5. Какие endpoints для student-данных (ФИО, класс, кампус, родители)?
   6. Можно ли подписаться на webhook'и при изменениях?
   7. Какие rate limits?

4. **Сделать пробный pull посещаемости за вчера** — для одного класса. Записать формат ответа в этот файл.

5. **Оценить scope для нашего привязывания ребёнок→семья:**
   - Toddle содержит родителей? Или только учеников?
   - Если содержит — мы переиспользуем это, или строим свою таблицу families в нашем backend и матчим по телефону родителя?
   - Это влияет на D-014 (data model source) — поэтому критично

6. **Эскалировать тревожные сигналы Bekhruz:**
   - Если у Toddle нет API → нужно решение: парсить UI? Получать ежедневный CSV-экспорт? Перенести attendance на v1.1?
   - Если посещаемость доступна с задержкой >1 часа → меняем UX-обещание родителю
