# Parent App — Низ воронки

## Что это
Веб/мобильное приложение для **текущих** родителей Оксбриджа. Низ воронки: здесь клиенты получают ту ценность, за которую платят. Начальный фокус — функции, которые быстрее всего окупаются в доверии и вовлечённости — оплата, посещаемость, питание, проходные, форма, сообщения.

**Архитектурный принцип: Parent App — это клиент, а не система-источник истины.** Он не владеет данными об учениках, оценках, посещаемости или финансах. Это живёт в системах-источниках истины. Parent App читает из этих систем и предоставляет цельный опыт для родителя.

**Системы-источники истины, из которых Parent App читает данные:**
- **Сегодня:** Toddle (ученики, классы, посещаемость, оценки, расписания), бухгалтерская программа 1C (балансы, начисления), Hikvision (события входа/выхода), школьные коммуникационные инструменты.
- **Завтра (2027–28 учебный год):** наша собственная LMS/SIS заменит Toddle для данных по ученикам. Parent App адаптируется к новым контрактам; роль потребителя не меняется.

Именно поэтому **Parent App и будущая LMS/SIS должны хорошо взаимодействовать.** Когда родитель хочет написать учителю, увидеть оценки или проверить домашнее задание, Parent App читает оттуда, где эти данные живут. Контракт между Parent App и системами-источниками — это первоклассный архитектурный артефакт. Стабильные контракты позволяют менять источники без поломки клиента.

Принимаем такой компромисс: сегодня мы строим интеграции с Toddle и бухгалтерией. Завтра мы перестраиваем эти интеграции, чтобы они смотрели на нашу LMS. UI Parent App не сильно меняется, но слой интеграции переписывается. Это известная, запланированная цена.

## Видение (длинная тень)
Parent App со временем становится более насыщенной поверхностью для родителей — лучше уведомления, больше интеграций (оплата, посещаемость, транспорт, коммуникации), более качественный UX. Он **не** пытается стать школьной информационной системой.

**Что Parent App должен делать к 15 августа:** быть незаменимым ежедневным инструментом для текущих родителей — оплата прежде всего (forcing function для сентябрьских платежей), затем посещаемость, питание, проходные, сообщения. Родители должны открывать его ежедневно без трения.

**Чем Parent App НЕ пытается стать:**
- Инструментом для учителей / администрации
- Журналом оценок или авторитетной системой посещаемости
- Финансовым back-office (он показывает баланс; бухгалтерия живёт в другом месте)
- LMS

**Смежная будущая работа (не входит в scope Parent App):**
- **LMS / SIS замена** — планируется на 2027–28 учебный год. Заменяет Toddle. Построена на Google fundamentals. Отдельная папка, когда этот проект начнётся.
- **Teacher App** — отдельная поверхность для учителей, скорее всего, появится после LMS.
- **Admin dashboard** — оперативные инструменты для управления школой.

Когда начнётся работа над LMS, мы решим, мигрирует ли Parent App на её модель данных или остаётся отдельной поверхностью, потребляющей API LMS. Это решение будет жить в Decision-файле в тот момент.
